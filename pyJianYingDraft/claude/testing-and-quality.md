# 测试与质量

## 验证方式

- **无自动化测试**。
- 手动验证：运行 `demo.py` / `demo_subdrafts.py`，在剪映中打开生成草稿检查时间轴。

## 质量风险

1. 无自动化测试，回归依赖人工。
2. 剪映版本升级使元数据 ID 失效，需从实际草稿更新 `metadata/`。
3. 时间单位（微秒 vs 秒）混淆是高频 bug。
4. 模板模式无法兼容剪映 6+ 加密。
5. 自动导出强依赖剪映 UI 控件，剪映改版易失效（≤6.x 才稳定）。
6. `uiautomation` 仅 Windows，跨平台代码需条件导入。

## 建议

- 改动 `metadata/` 或片段导出后，至少用一个真实草稿在剪映中验证。
- 改动 `time_util.py` 后检查所有调用点的时间单位。
